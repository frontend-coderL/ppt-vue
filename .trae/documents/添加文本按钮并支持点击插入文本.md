## 功能概述
- 在每个 `SlideEditor` 的工具栏新增“文本”按钮。
- 点击“文本”按钮后进入一次性插入模式；在当前激活页的 iframe 画布内单击，将在点击位置新增一个文本元素，默认内容为“双击可编辑”。
- 新增元素可被选中并通过 Moveable 拖拽/缩放；双击可进入内联文本编辑（已有逻辑）。

## 交互流程
1. 用户点击“文本”按钮（仅对激活页生效）。
2. 鼠标在该页 iframe 画布内单击：
   - 计算点击点在未缩放内容坐标系中的位置（考虑 `fitIframeContent` 对 `body` 的缩放）。
   - 在该位置创建一个 `div` 文本元素，内容为“双击可编辑”。
   - 自动选中该元素并绑定 Moveable。
   - 插入模式自动退出（一次性）。

## 技术实现
- 在 iframe 的 `window.__editorState` 中加入一次性插入标记：`pendingInsert: 'text' | null`。
- 暴露新方法 `armTextInsert(frame)`：
  - 设置 `pendingInsert = 'text'`。
  - 可选：将 `doc.body` 的光标设置为 `text`，提示插入模式；插入完成后恢复。
- 修改 `pointerdown` 处理（优先判断插入模式）：
  - 若 `pendingInsert === 'text'`：
    - 读取 `body` 的缩放比例：从 `getComputedStyle(body).transform` 的矩阵提取 `scaleX/scaleY`（无缩放时默认 1）。
    - 使用 `clientX/clientY` 与 `body.getBoundingClientRect()` 计算点击点在内容坐标系中的 `(x, y)`：`(client - rect) / scale`。
    - 创建元素：`div`，`textContent = "双击可编辑"`，样式：`position: absolute; transform: translate(xpx, ypx); font-size: 16px; color: #333;` 等基础样式。
    - `body.appendChild(el)`，调用 `selectTarget(el)`。
    - 清空 `pendingInsert` 并恢复光标。
  - 否则维持原有选中逻辑。
- 在 `SlideEditor.vue` 中新增“文本”按钮与处理函数：
  - `function insertText()`：若是激活页且存在 `iframeRef`，调用 `armTextInsert(iframeRef.value)`。

## 兼容与边界
- 缩放适配：通过解析矩阵确保点击位置准确映射到内容坐标。
- 只对激活页生效：依赖已有 `active` 逻辑（`src/components/SlideEditor.vue:44-54`）。
- 与 Moveable 交互：保持对 `.moveable-control, .moveable-line` 的点击避让（已有逻辑）。
- 多页并存：插入标记保存在各自 iframe 的 `window.__editorState`，互不影响。

## 验证步骤
- 本地运行后在激活页：
  - 点击“文本”，在不同位置单击，元素应出现在点击点。
  - 双击新元素进入编辑，输入自定义文本，`Enter/Escape` 退出正常。
  - 拖拽/缩放新元素正常。
  - 再次点击“文本”再次插入，模式一次性结束。
- 切换到非激活页后，按钮不应影响其他页（或仅在激活页工作）。
